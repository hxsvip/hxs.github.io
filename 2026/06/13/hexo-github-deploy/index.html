Hexo 搭建的静态网站如何部署到个人 GitHub 上，以及如何实现内容更新并推送到 GitHub

文章写好了，怎么让更多人看到？这篇教程手把手教你把 Hexo 博客部署到 GitHub Pages，并掌握日常更新推送的完整流程。

前言

在前两篇文章中，我们已经完成了 Hexo 框架的安装配置（第一篇）和 AnZhiYu 主题的配置（第二篇）。现在你的博客已经在本地完美运行了，但要想让全世界都能看到你的博客，就需要把它部署到网上。

GitHub Pages 是一个免费、稳定的静态网站托管服务，配合 Hexo 使用简直天作之合。本文将详细介绍部署的完整流程。

第一部分：准备工作

1. 注册 GitHub 账号

如果你还没有 GitHub 账号，请访问 https://github.com 注册一个。

2. 创建代码仓库

部署到 GitHub Pages 需要创建一个以 用户名.github.io 命名的仓库。

  1. 登录后点击右上角的 + 号，选择 New repository
  2. 在 Repository name 处填写：你的用户名.github.io（例如你的用户名是 hxsvip，则填写 hxsvip.github.io）
  3. 设置为 Public（公开仓库）
  4. 不需要初始化 README、.gitignore 或 license
  5. 点击 Create repository

💡 为什么要用 用户名.github.io 这个命名规则？
因为 GitHub 有一个用户/组织 Pages 服务规则：只有名为 用户名.github.io 的仓库才会被自动识别为个人站点。

1. 修改 _config.yml

字段说明：

字段 说明
type 部署类型，使用 git
repo 远程仓库地址，使用 SSH 格式（git@github.com:用户名/仓库名.git）
branch 推送到哪个分支，GitHub Pages 推荐使用 main 或 master

💡 如果你的仓库是 HTTPS 地址，也可以写 https://github.com/hxsvip/hxsvip.github.io.git，但每次部署都需要输入用户名和密码，不太方便。推荐使用 SSH 方式。

第三部分：首次部署

完整的部署流程

将本地博客推送到 GitHub Pages 需要执行以下四个命令：

命令执行顺序的详细说明：

步骤 命令 作用
① hexo clean 清除 public/ 和 db.json 缓存文件
② hexo generate 将 source/_posts/ 下的 Markdown 文件编译为静态 HTML，生成到 public/ 目录
③ hexo deploy 将 public/ 目录的内容推送到 GitHub 仓库

部署成功后的效果

部署成功后，在浏览器中访问 https://你的用户名.github.io，就能看到你的博客了！

🎉 第一次部署可能需要等待 1~5 分钟，GitHub 需要时间构建和分发你的站点。如果访问时显示 404，请稍等片刻再刷新。

常见问题排查

问题 1：部署时报 Authentication failed

原因： SSH 密钥未正确配置或仓库地址错误。

解决方法：

  1. 确认 SSH 密钥已添加到 GitHub：ssh -T git@github.com
  2. 检查 _config.yml 中 deploy.repo 的地址是否正确
  3. 如果是第一次部署，可能需要手动添加远程仓库

问题 2：访问站点显示 404

原因： GitHub Pages 构建需要时间，或者仓库名不正确。

解决方法：

  1. 等待 5~10 分钟后再访问
  2. 确认仓库名为 用户名.github.io（全小写）
  3. 进入仓库的 Settings → Pages，查看构建状态

问题 3：样式丢失或图片无法显示

原因： 相对路径问题或静态资源未正确生成。

解决方法：

  1. 执行 hexo clean && hexo generate 重新生成
  2. 检查 public/ 目录下是否有 css/、js/、img/ 等文件夹
  3. 确认 _config.yml 中的 url 和 root 配置正确

第五部分：GitHub Pages 高级配置

1. 自定义域名

如果你想用自己的域名（如 www.example.com）访问博客：

在 GitHub 上配置

  1. 进入你的仓库 Settings → Pages
  2. 在 Custom domain 处填写你的域名（如 www.example.com）
  3. 勾选 Enforce HTTPS
  4. 点击 Save

添加 DNS 记录

在你的域名服务商处添加以下记录：

类型 名称 值
CNAME www 你的用户名.github.io

或者添加 A 记录指向 GitHub Pages 的 IP：

类型 名称 值
A @ 185.199.108.153
A @ 185.199.109.153
A @ 185.199.110.153
A @ 185.199.111.153

2. 开启 HTTPS

GitHub Pages 默认提供 HTTPS 支持。在仓库 Settings → Pages → Enforce HTTPS 勾选即可。

3. 发布状态监控

在仓库 Settings → Pages 页面，你可以看到：

  • 当前站点的构建状态（Success / Failed）
  • 最近一次部署的时间
  • 部署时的错误信息

如果构建失败，点击 View deployment 可以看到详细的构建日志。

第六部分：其他部署方式（可选了解）

Vercel 部署

Vercel 也支持静态网站托管，部署流程类似：

  1. 在 vercel.com 注册账号
  2. 导入你的 Hexo 仓库
  3. 设置构建命令为 hexo generate
  4. 设置输出目录为 public

Cloudflare Pages

  1. 在 cloudflare.com 注册
  2. 连接 GitHub 仓库
  3. 框架预设选择 Hexo
  4. 构建命令：hexo generate
  5. 输出目录：public

对比

部署方式 费用 速度 自定义域名 推荐度
GitHub Pages 免费 国内较慢 支持 ⭐⭐⭐⭐
Vercel 免费 快 支持 ⭐⭐⭐⭐⭐
Cloudflare Pages 免费 快 支持 ⭐⭐⭐⭐⭐
阿里云 OSS 低成本 快 支持 ⭐⭐⭐⭐

💡 国内访问建议： GitHub Pages 在国内访问速度较慢，如果对访问速度有要求，推荐使用 Vercel 或 Cloudflare Pages。但 GitHub Pages 适合学习和入门，零成本零门槛。

写在最后

到这里，你已经掌握了 Hexo 博客从本地搭建到线上部署的完整流程：

  1. ✅ 创建 GitHub 仓库（用户名.github.io）
  2. ✅ 配置 SSH 密钥
  3. ✅ 安装部署插件 hexo-deployer-git
  4. ✅ 修改 _config.yml 部署配置
  5. ✅ 执行 hexo clean && hexo deploy -g 完成首次部署
  6. ✅ 日常更新时重复生成+推送流程

记住这个核心命令：hexo d -g，写完后执行它，你的博客就上线了！

如果部署过程中遇到任何问题，欢迎回来翻看这篇教程，或者在 GitHub 上搜索相关错误信息。
